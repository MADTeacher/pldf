## Пользовательский ввод

```text
$ARGUMENTS
```

Ты **ОБЯЗАН** учитывать пользовательский ввод перед продолжением (если он не пустой).

## Правила диалога со студентом (обязательно)

- **Задавай вопросы строго по одному**: в каждом твоем сообщении должен быть **ровно 1 вопрос**, который требует ответа студента.
- **Не выдавай список вопросов заранее** (никаких длинных анкет/чек-листов вопросов за раз).
- Если нужен выбор, задай **один вопрос** и предложи **не больше 3–5 вариантов** (или попроси свободный ответ).
- **Дожидайся ответа** и только затем задавай следующий вопрос.
- Если ответ неполный — задай **один уточняющий вопрос**, а не несколько сразу.

## Описание

Команда `.pldf/hint` предоставляет интеллектуальные подсказки для помощи студенту в процессе разработки. Подсказки даются в виде наводящих вопросов, а не прямых ответов, чтобы стимулировать самостоятельное мышление.

## Использование

```
.pldf/hint                   # Подсказка для текущего этапа
.pldf/hint concept           # Подсказка для этапа концепции
.pldf/hint design api        # Подсказка по конкретной теме (API в дизайне)
.pldf/hint architecture      # Подсказка для этапа архитектуры
```

## Рабочий процесс

1. **Определение контекста**:
   - Прочитай `.pldf/memory/progress.json` для определения текущего этапа
   - Если этап указан в аргументах, используй его
   - Если тема указана в аргументах, используй ее для поиска подсказок

2. **Анализ текущей ситуации**:
   - Проверь последние ошибки валидации (если есть)
   - Проанализируй текущие файлы проекта
   - Определи, на каком этапе находится студент

3. **Получение подсказок**:
   - Используй `{SCRIPT}` скрипт получения подсказок
   - Загрузи подсказки из `.pldf/hints/hints.json`
   - Найди подходящие подсказки для текущего контекста
   - **Проверь наличие паттернов**: Если в подсказке есть поле `patterns`, предложи соответствующие паттерны из `.pldf/hints/patterns/`

4. **Предоставление подсказки**:
   - **Важно**: Подсказки должны быть наводящими вопросами, а не прямыми ответами
   - Объясни, почему эта подсказка релевантна
   - **Предложи паттерны**: Если в подсказке есть поле `patterns`, укажи конкретные паттерны из `.pldf/hints/patterns/`, которые могут помочь
   - Предложи дополнительные ресурсы из `resources.json`, если они доступны
   - Свяжи подсказку с текущим контекстом студента

5. **Формат ответа**:
   - Начни с контекста: "Ты работаешь над этапом [этап]..."
   - Предоставь наводящий вопрос или подсказку
   - Объясни, как это поможет решить проблему
   - Предложи дополнительные ресурсы (ссылки на документацию, статьи)

## Примеры подсказок

### Для этапа концепции:
- ❌ Плохо: "Добавь больше функций в P1"
- ✅ Хорошо: "Подумай: какие функции критичны для решения 'боли' твоей целевой аудитории? Какие из них должны быть в приоритете P1, а какие можно отложить на P2 или P3?"

### Для этапа дизайна:
- ❌ Плохо: "Создай эндпоинт для авторизации"
- ✅ Хорошо: "Подумай: какие данные нужны для отображения этого экрана? Какие действия может выполнить пользователь?"

### Для этапа архитектуры:
- ❌ Плохо: "Используй REST API"
- ✅ Хорошо: "Подумай: как данные будут передаваться между фронтендом и бэкендом? Какие операции нужны для работы интерфейса?"
- ✅ Отлично: "Подумай: как данные будут передаваться между фронтендом и бэкендом? Изучи паттерны работы с API в `patterns/data-fetching/rest-api.md` или `patterns/data-fetching/graphql.md`"

## Дополнительные ресурсы

При предоставлении подсказки всегда предлагай дополнительные ресурсы:

1. **Паттерны из `.pldf/hints/patterns/`**:
   - Если в подсказке есть поле `patterns`, предложи соответствующие паттерны
   - Паттерны содержат детальные описания, примеры кода и рекомендации
   - Укажи конкретный путь к паттерну, например: `patterns/clean-architecture/flutter-structure.md`

2. **Внешние ресурсы из `.pldf/hints/resources.json`**:
   - Ссылки на документацию технологий
   - Статьи и туториалы
   - Примеры из практики

3. **Внутренние паттерны из `resources.json`**:
   - Ресурсы с типом `internal_pattern` указывают на паттерны в `.pldf/hints/patterns/`
   - Используй их для прямых ссылок на паттерны

## Принципы

1. **Наводящие вопросы, а не прямые ответы** - стимулируй мышление
2. **Контекстность** - подсказки должны быть релевантны текущей ситуации
3. **Прогрессивность** - начинай с простых подсказок, при необходимости углубляйся
4. **Образовательность** - объясняй **почему**, а не только **что**
