# Паттерны аутентификации и авторизации

Эта категория содержит паттерны для реализации аутентификации и авторизации в приложениях.

## Доступные паттерны

### [JWT токены](jwt-auth.md)
Использование JSON Web Tokens для stateless аутентификации. Подходит для REST API, SPA и мобильных приложений.

### [Сессионная аутентификация](session-auth.md)
Традиционный подход с серверными сессиями. Подходит для веб-приложений с серверным рендерингом.

### [OAuth 2.0](oauth.md)
Интеграция с внешними провайдерами (Google, GitHub, Facebook и т.д.). Подходит для упрощения регистрации пользователей.

## Выбор паттерна

### Используйте JWT, если:
- Разрабатываете REST API
- Нужна stateless аутентификация
- Приложение масштабируется горизонтально
- Есть несколько клиентов (web, mobile)

### Используйте сессии, если:
- Веб-приложение с серверным рендерингом
- Нужен контроль над сессиями на сервере
- Требуется возможность мгновенного разлогинивания

### Используйте OAuth, если:
- Хотите упростить регистрацию
- Нужна интеграция с социальными сетями
- Пользователи уже имеют аккаунты в популярных сервисах

## Общие принципы безопасности

1. **Никогда не храните пароли в открытом виде** - используйте хеширование (bcrypt, argon2)
2. **Используйте HTTPS** для всех запросов с аутентификацией
3. **Ограничивайте частоту запросов** (rate limiting) для предотвращения брутфорса
4. **Валидируйте токены** на сервере, не доверяйте клиенту
5. **Используйте короткое время жизни токенов** и refresh tokens для обновления
