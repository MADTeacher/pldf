# Паттерны получения данных

Эта категория содержит паттерны для работы с данными в клиентских приложениях: запросы к API, обработка ответов, кэширование и синхронизация.

## Доступные паттерны

### [REST API](rest-api.md)
Работа с REST API: создание клиентов, обработка запросов, обработка ошибок, кэширование.

### [GraphQL](graphql.md)
Работа с GraphQL: создание запросов, мутации, подписки, использование клиентов.

### [WebSocket](websocket.md)
Real-time коммуникация через WebSocket: установка соединения, обработка сообщений, переподключение.

## Выбор паттерна

### Используйте REST API, если:
- Простая структура данных
- Стандартные CRUD операции
- Команда знакома с REST
- Нужна простота и предсказуемость

### Используйте GraphQL, если:
- Сложные связанные данные
- Нужна гибкость в запросах
- Хотите уменьшить количество запросов
- Готовы к более сложной настройке

### Используйте WebSocket, если:
- Нужны real-time обновления
- Чат, уведомления, live данные
- Двусторонняя коммуникация
- Постоянное соединение оправдано

## Общие принципы

1. **Обработка ошибок**: Всегда обрабатывайте ошибки сети и API
2. **Загрузка состояния**: Показывайте индикаторы загрузки
3. **Кэширование**: Кэшируйте данные, когда это возможно
4. **Оптимистичные обновления**: Обновляйте UI до получения ответа сервера (где уместно)
5. **Retry логика**: Повторяйте запросы при временных ошибках
